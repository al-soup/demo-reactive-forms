Template Driven Forms

  • Easy to use
  • Suitable for simple scenarios and fails for complex scenarios
  • Similar to AngularJS
  • Two way data binding(using 
    [(NgModel)]
    syntax)
  • Minimal component code
  • Automatic track of the form and its data(handled by Angular)
  • Unit testing is another challenge

Name

Reactive Forms (Model Driven Forms)

  • More flexible, but needs a lot of practice
  • Handles any complex scenarios
  • No data binding is done (immutable data model preferred by most developers)
  • More component code and less HTML markup
  • Reactive transformations can be made possible such as
    • Handling a event based on a debounce time
    • Handling events when the components are distinct until changed
    • Adding elements dynamically
  • Easier unit testing

Name

The Mix

  • "If by some reason we would need to, we could mix and match the two ways of building forms,...(Formgroup Validations & ngModel)"
  • "...But in general, it's better to choose one of the two ways of doing forms, and using it consistently throughout the appliation." (Angular-University)
